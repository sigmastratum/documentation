# Contribution Process

Contributions to the Sigma Runtime Standard are welcomed through the SRIP
process.

### How to Contribute

1. Review existing SRIPs and documentation.
2. Open an issue or discussion in the SRS Git repository.
3. Submit an SRIP draft or patch via pull request.
4. The Steering Committee reviews the proposal.
5. If accepted, the change becomes part of the next SRS release.

Contributors must agree to the project's licensing and IP policies.
