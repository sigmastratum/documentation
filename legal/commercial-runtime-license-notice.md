Open standard. Closed runtime. Governed certification.

## Short Rule

You can build an independent implementation of the public SRS/SRIP standard without a Sigma commercial runtime license.

You need separate written terms when you use Sigma Runtime as a product or service, Sigma proprietary assets, Sigma marks, official certification, managed Sigma deployment, white-label deployment, resale, or CC BY-NC materials commercially.

## Examples

| Use case | Sigma commercial runtime license required? |
|---|---|
| Reading SRS/SRIP | No |
| Citing SRS/SRIP with attribution | No |
| Independent implementation of public SRS/SRIP requirements | No |
| Selling an independent product built from public SRS/SRIP requirements without Sigma marks/certification | No |
| Using Sigma Runtime proprietary code or private modules | Yes |
| Accessing private ALICE modules | Yes |
| White-label Sigma Runtime deployment | Yes |
| Reselling Sigma Runtime access | Yes |
| Claiming Sigma Certified status | Yes |
| Using `∿` as a product badge or certification mark | Yes |
| Commercializing CC BY-NC research-origin materials | Yes, unless separately authorized |

## Product Boundary

Sigma Runtime as an operated product includes sigmaruntime.com, proprietary runtime code, private modules, ALICE internals, runtime telemetry, production memory/control mechanisms, private datasets, product UX, commercial deployment topology, support, SLA, and enterprise deployment assets.

Those product assets are separate from the public SRS/SRIP specification layer.
